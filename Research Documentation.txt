# A Domain-Specific LLM for Cyber Threat Hunting Based on LLAMA-3 and MITRE TTPs

Project Overview
This research project develops a domain-specific Large Language Model (LLM) for cyber threat hunting, leveraging LLAMA-3.1 8B and MITRE ATT&CK TTPs (Tactics, Techniques, and Procedures). This repository contains the complete implementation and dissertation for PhD research on developing a domain-specific Large Language Model (LLM) for cyber threat hunting, focusing on Novel TTP Discovery rather than classification.

### Research Contributions
1. Novel TTP Discovery Framework - First systematic approach to generating genuinely new attack techniques
2. Custom Multi-Dimensional Reward - Balanced evaluation across relevance, novelty, impact, and feasibility
3. Adversarial Robustness Training - Integrated perturbation-based training with 97.72% defense rate
4. Curriculum Learning Strategy - Gradual difficulty increase (5% → 15% perturbation ratio)
5. Dynamic λ Adaptation - Automatic adversarial penalty adjustment.

Module Descriptions
1. Data Preprocessing (01_data_preprocessing.py)
•	Input: MITRE ATT&CK v18.0 Enterprise data
•	Output: Structured instruction-response pairs in JSON format
•	Split: 80% training, 20% testing (stratified by tactic)
2. Synthetic Data Generation (02_synthetic_data_generation.py)
•	Methodology: Evol-Instruct (adapted from WizardLM)
•	Expansion Factor: 5x original training data
•	Strategies:
–	Stylistic variations (40%)
–	Contextual augmentation (30%)
–	Parameter variation (20%)
–	Technique combination (10%)
3. Custom Reward Function (03_reward_function.py)
The research introduces a novel multi-dimensional reward function.
Mathematical formulation:
R_total = α₁·R_relevance + α₂·R_novelty + α₃·R_impact + α₄·R_feasibility

Where:
- α₁ = 0.30 (Relevance weight) Relevance to MITRE taxonomy
- α₂ = 0.25 (Novelty weight) Novelty compared to known TTPs
- α₃ = 0.30 (Impact weight) Security impact severity
- α₄ = 0.15 (Feasibility weight) Operational feasibility

Target scores:
- Relevance: >0.8
- Novelty: >0.7
- Impact: >0.6
- Feasibility: >0.6
- Overall: >0.75

4. Adversarial Perturbation (04_adversarial_perturbations.py)
•	FGSM: Fast Gradient Sign Method for text perturbation
•	PGD: Projected Gradient Descent for iterative attacks
•	Backtranslation: Semantic-preserving paraphrase generation
•	Robustness Penalty: λ × std_dev(perturbed_scores)
5. Model Training (05_model_training.py)
•	Base Model: Llama 3.1 8B Instruct
•	Quantization: QLoRA (4-bit with LoRA rank 64-128)
•	Optimizer: GRPO (Group Relative Policy Optimization)
•	Curriculum Learning: Perturbation ratio 5% → 15%
•	Dynamic λ Adaptation: Double if ASR > 10%, cap at 0.3
6. Evaluation (06_evaluation.py)
Comprehensive evaluation metrics: - Relevance Score (MITRE alignment) - Novelty Score (compared to known TTPs) - Impact Score (security severity) - Feasibility Score (operational practicality) - Attack Success Rate (ASR) - Statistical significance testing

## GRPO Training Details

### Group Relative Policy Optimization (GRPO)

GRPO extends standard policy optimization by incorporating group-based reward normalization:

| GRPO Parameter | Value |
|----------------|-------|
| Group Size | 8 |
| KL Coefficient (β) | 0.1 |
| Clip Range | 0.2 |
| Value Function Coefficient | 0.5 |
| Entropy Coefficient | 0.01 |
| Max Grad Norm | 1.0 |
| GRPO Iterations | 1000 |
| Reward Baseline | Group Mean |

### Multi-Dimensional Reward Function

The reward function combines four components:

```
R_total = w1·R_semantic + w2·R_tactical + w3·R_novelty + w4·R_coherence
```

| Reward Component | Weight | Description |
|------------------|--------|-------------|
| Semantic Similarity (R_semantic) | 0.3 | Cosine similarity to known TTP embeddings |
| Tactical Alignment (R_tactical) | 0.3 | Alignment with MITRE ATT&CK tactics |
| Novelty Score (R_novelty) | 0.2 | Divergence from existing techniques |
| Coherence Score (R_coherence) | 0.2 | Linguistic and structural quality |
